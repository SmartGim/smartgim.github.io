从零单排技术之环境搭建

发表于 2020-04-05 | 分类于 技术

为什么需要自己的一台服务器

在公司的时候我们往往依赖运维提供给我们的环境去做开发，如果自己在家或者自己只有一个笔记本的话，想随时模拟出来一套功能完善的环境实际是很麻烦的，所以为了方便自己的学习，我自己组装了一台机器，安装了windows 和 Ubuntu 双系统（后面会讲解组装的配件购买和安装过程），平时自己使用 Mac 笔记本连接控制。Ubuntu上通过docker 或者 直接安装 模拟了实际开发中需要的一些软件。有人说为啥不买一台云服务呢，原因如下：

  • 云服务器性能低，好的太贵，不值得
  • 自己的服务器想怎么玩就怎么玩，不用担心以后续费的问题
  • 通过安装 Frp,或者花生壳可以很方便的把家里的机器变成云服务器
  • 组装电脑也是一种乐趣，并且拥有双系统还能娱乐
  • 如果一个软件自己都没有亲自安装过，实践过，说掌握了那个技术未免牵强

现在的互联网开发一般都需要什么

现在互联网虽然分工比较细，各有各自的职责，但是一些软件在大小公司都是通用的，我们以目前流行的前后端分离项目，并且微服务为例，通常互联网公司使用的工具如下

持续集成工具 Jenkins

前端打包工具 Nodejs

后端私服 Nexus

后端打包工具 Maven

注册中心 nacos 或者 consul,Eureka

负载均衡 nginx

系统监控工具 Prometheus ，Pushgateway，Grafana

微服务监控 SkyWalking

分布式协调 Zookeeper

消息队列 Kafka ，RocketMq

缓存 Redis

文档 JIRA

数据库 Mysql ，Oracle

搜索集成 ELK

自己搭建的 Gitlab 或者 Bitbucket 仓库

其他测试或者协同工具

其实常用的也就这些，自己没事的时候这些都可以自己搞定，顺便学习下运维，自己安装的还能随时查看性能指标之类的东西。软件的安装一定要查看官方的手册。自己动手实践非常重要，不要光看教程，教程只是参考而已。
